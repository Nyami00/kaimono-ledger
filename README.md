# 買い物台帳（kaimono-ledger）

レシートを記録し、購入間隔と価格から「次の買い時」を提案する**端末内完結**のPWA。
購入データはスマホのブラウザ内（IndexedDB）にのみ保存され、サーバーに送られず、他人からは見えません。
APIやローカルLLM（Ollama）は使いません。レシート読み取りは**サブスクのチャット**または**端末の文字認識**の出力を貼り付けて取り込みます。

## 🚀 起動URL

**<https://nyami00.github.io/kaimono-ledger/>**

スマホのブラウザで開き、「ホーム画面に追加」でアプリとして使えます（[手順](#2-スマホにアプリとして追加)）。
※ GitHub Pages を有効化していない場合は、下の[公開手順](#1-github-pagesで公開)で発行してください（公開後、上記URLで起動します）。

## 1. GitHub Pagesで公開

1. GitHubで新規リポジトリ（例 `kaimono-ledger`）を作成。Private/Publicどちらでも可。
2. 上記4ファイルを**リポジトリ直下**に置いてpush。
3. **Settings → Pages** → Source: `Deploy from a branch` → Branch: `main` / `/ (root)` → Save。
4. 1〜2分後、`https://<ユーザー名>.github.io/kaimono-ledger/`（このリポジトリでは <https://nyami00.github.io/kaimono-ledger/>）が発行されます。

> **データは端末内**：コード（URL）が公開でも、購入データは各自のブラウザに隔離されます。他人が同じURLを開いても見えるのは空の画面だけです。URL自体も隠したいなら、`index.html`をスマホに保存してローカルで開く運用でも動きます。

## 2. スマホにアプリとして追加

- 発行URLをスマホのブラウザで開く。
- **iPhone（Safari）**：共有 → 「ホーム画面に追加」。
- **Android（Chrome）**：メニュー → 「アプリをインストール / ホーム画面に追加」。
- 以後はホームのアイコンから起動。オフラインでも全機能が動きます。

## 3. レシートの読み取り（2つの方法・どちらもサブスク内/無料）

> APIは使わないため、アプリから直接LLMは呼びません。**チャットや端末OCRの出力を「記録」タブに貼り付け**て取り込みます。

### 方法A：チャットで自動抽出（高精度・推奨）

1. アプリ「設定」タブの **「📋 プロンプトをコピー」** で読み取り用プロンプトを取得。
2. それを **Claudeのプロジェクト**（カスタム指示）または **ChatGPTのカスタムGPT** に登録。
   - 一度登録すれば、以後はレシート写真を貼るだけで定型JSONが返ります。
3. レシート写真をそのチャットに貼る → 返ってきた **JSONをコピー**。
4. アプリ「記録」タブ①の欄に**貼り付け → 「明細を取り込む」**。
5. 抽出明細をチェック・価格修正して **「まとめて記録」**。店舗・日付は自動入力されます。

### 方法B：端末の文字認識（オフライン・補助）

1. スマホでレシートを撮影し、**iPhoneのテキスト認識表示**／**AndroidのGoogleレンズ**で文字を選択してコピー。
2. アプリ「記録」タブ①の欄に**貼り付け → 「明細を取り込む」**。
3. 行を自動分解（合計・小計・お預り・お釣り・日付などは自動除外）。チェック・修正して記録。

いずれも使わず**手入力**だけでも全機能が利用できます（②のフォーム）。

## 機能メモ

- **買う候補（4回以上）**：「候補」タブに、同一商品名を**4回以上**記録した「よく買うもの」を一覧表示。購入回数・前回購入日・平均間隔・目安単価・よく買う店を表示し、買い時が近い順（購入時期→そろそろ→様子見）に並びます。
- **業態で選ぶ**：「候補」タブ上部の**業態ボタン**（スーパー／ドラッグストア／コンビニ／ホームセンター／百貨店・モール／ネット通販／その他）を押すと、その業態の店で買うものだけに絞り込み。もう一度押すと解除。これから行く店の種類を選ぶだけで、その買い候補が出ます。
- **業態の記録**：レシート取り込み用プロンプトに `storeType`（業態）を追加。チャットがレシートから業態を自動判定してJSONに含め、取り込み時に自動セットされます。手入力時は②のフォームの「業態」から選択。過去の業態未登録データは、主要チェーンの店名から自動推定して業態ボタンに反映します（CSV/JSONにも `業態`/`storeType` 列を追加）。
- **間隔分析**：同一商品名（NFKC正規化＋空白除去で名寄せ）を2回以上記録すると、`前回購入日 + 平均間隔`で**次回予測日**を算出。予測日に近い／過ぎた品目を「分析」タブの**そろそろ買う候補**に表示。
- **価格分析**：品目ごとに単価（価格÷数量）の直近・平均・最安・最高と、平均比トレンド（▲高め/→平均的/▼安め）を表示。
- **バックアップ**：CSV（Excel対応の**BOM付きUTF-8**）/ JSON書き出し、JSONから復元。機種変更前にJSONを保管してください（画像以外を完全復元）。
- **名寄せの注意**：「牛乳」と「牛乳 1L」は別品目になります。間隔を見たい商品は表記を揃えると精度が上がります。

## 拡張のヒント

- **複数端末で共有したい**：書き出したJSONを自分のNotion/Driveに保管して取り込む運用、または保存先をNotion DB等へ切り替える改修で対応可能。
